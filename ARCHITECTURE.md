# Архитектура

## Стек

- **Next.js 15** (App Router, TypeScript) — UI мини-аппа + API-роуты. Деплой — Vercel.
- **@telegram-apps/sdk-react** — оболочка мини-аппа; `initData` валидируется на сервере (HMAC по
  токену бота), не на клиенте.
- **Supabase** (Postgres) + **pgvector** — основные данные и векторная база знаний.
- **Anthropic API** — `claude-sonnet-4-6` для диалога (paid), `claude-haiku-4-5` для кризисного
  классификатора и суммаризации истории (и для диалога на free-тарифе). Prompt caching на
  системный промпт и стабильные блоки KB.
- **grammY** (Node) — бот: запуск мини-аппа, инвойсы Telegram Stars, вебхук оплаты.
- **bun** — пакетный менеджер и runtime для разработки.

Все ключи API — только на сервере (`.env`, не коммитится). Любые обращения к LLM — через
серверные роуты `/app/api/*`, никогда напрямую с клиента.

## Эмбеддинги для RAG: Voyage AI (`voyage-multilingual-2`)

Выбран как временное бесплатное решение:
- официально рекомендован Anthropic как embedding-партнёр;
- мультиязычная модель с качественной поддержкой русского;
- бесплатный trial-тир (≈200M токенов) — достаточно для прототипа и первой загрузки базы знаний.

Слой RAG (`/lib/rag`) обращается к эмбеддингам через единый интерфейс
(`embedText(text): Promise<number[]>`), без завязки кода ретривала/ингеста на конкретного
провайдера — при необходимости Voyage можно заменить на другого провайдера, поменяв только
реализацию этой функции.

## Структура репозитория

```
/app                 — Next.js роуты (мини-апп UI + /api)
/app/api/chat        — прокси к Anthropic + RAG + safety
/app/api/telegram    — вебхук оплаты Stars
/lib/llm             — клиенты Anthropic, prompt caching, суммаризация истории
/lib/safety          — классификатор кризиса + кризисный протокол
/lib/rag             — ретривал, ингест, интерфейс эмбеддингов
/lib/rag/ingest-source — сюда владелец кладёт разрешённые материалы для ингеста (см. ниже)
/lib/billing         — квоты, Stars-инвойсы
/config/modules      — конфиги модулей/подходов
/bot                 — grammY: запуск аппа, инвойсы
/db                  — миграции Supabase + pgvector
```

## Модули/подходы (конфиг, не код)

Каждый модуль или терапевтический подход — запись конфига:

```json
{
  "id": "burnout_match",
  "title": "Спичка",
  "system_prompt": "...",
  "allowed_techniques": ["..."],
  "kb_namespace": "burnout",
  "ui_accent": "...",
  "approach": "cbt|psychodynamic|gestalt|mix|art"
}
```

Добавление нового модуля = новая запись конфига, без изменения кода диалогового пайплайна. Для
прототипа — 2 реальных модуля («Спичка» + один подход), остальные — заглушки в маркетплейсе.

## Safety-слой (P0, встроен в основной чат-пайплайн с самого начала)

Перед генерацией обычного ответа сообщение пользователя проходит классификатор на
`claude-haiku-4-5`: `normal` / `crisis` (суицидальные мысли, самоповреждение, угроза другим,
острый кризис, насилие).

- При `crisis` — обычный диалог не продолжается. Возвращается кризисный протокол: валидация
  чувств, выражение заботы, направление к экстренной помощи и живому человеку, вывод проверенных
  контактов.
- Контакты — конфиг `crisis_resources` (плейсхолдеры, реальные номера вписывает владелец из
  официальных источников).
- Системный промпт включает жёсткие рамки: без диагнозов, без рекомендаций по препаратам/дозам,
  без заявлений о замене терапии, направление к специалисту.
- Срабатывания классификатора логируются и помечаются для ручного разбора.

## База знаний (RAG)

- Скрипт ингеста: материалы из `/lib/rag/ingest-source` → чанкинг → эмбеддинги (Voyage) →
  pgvector, с метаданными (`kb_namespace`, источник, подход).
- Выдача: top-k по namespace модуля, инъекция в контекст. Стабильная системная часть — в
  prompt cache.
- Копирайт: в базу попадают только материалы, на которые у владельца есть права (собственные
  тексты, конспекты своими словами, public domain, лицензированный контент). Коммерческие книги
  целиком — не загружать.

## Экономия токенов в рантайме

- Prompt caching на системный промпт + конфиг модуля + стабильные блоки KB.
- Суммаризация истории на Haiku при превышении окна контекста.
- Free-тариф → `claude-haiku-4-5`; paid → `claude-sonnet-4-6`, выбор модели по тарифу пользователя.
- Учёт токенов на каждый запрос в таблице `usage`.

## Дизайн-система

Эстетика — редакционный премиум-минимализм (референс Goldapple), палитра — фиолетовая.

MCP `styles.refero.design` в этом окружении не подключён (доступны только MCP `figma` и
`higgsfield`), поэтому токены задаются вручную как фолбэк-палитра, без автоматического вытягивания
референса:

- тёмный сливово-фиолетовый фон;
- поверхности — тёмный violet;
- акцент — насыщенный фиолетовый (`#7C3AED` / `#8B5CF6`);
- мягкий лиловый и тёплый off-white для текста;
- градиентные акценты, «стеклянные» карточки, крупные заголовки.

Все цвета — через CSS-переменные/токены, не хардкодить по месту.

## Схема БД (Supabase, детали — в Фазе 1)

Таблицы: `users`, `conversations`, `messages`, `usage`, `subscriptions`, `modules`. Векторная
таблица для базы знаний — через расширение `pgvector`.
